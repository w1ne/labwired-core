- [ ] **Step 4: Add the SR dispatch enum and probe to `openocd.rs`**

Insert the following just above the existing `read_register` method (around line 178 of `crates/hw-oracle/src/openocd.rs`):

```rust
    // ── SR access form dispatch (Plan 1.5 fix #1) ─────────────────────────────

    /// One of three OpenOCD command forms for accessing an Xtensa SR.
    ///
    /// The ESP32-S3 LX7 target exposes some SRs only via `xtensa rsr <name>`;
    /// others are exposed as ordinary `reg <name>` aliases.  We probe each form
    /// once per session per SR and cache the working one.
    #[derive(Debug, Clone, Copy, PartialEq, Eq)]
    pub enum SrAccessForm {
        /// `reg <name>` — works for general-purpose registers and most SRs.
        Reg,
        /// `xtensa rsr <name>` / `xtensa wsr <name> <val>` — Xtensa-specific.
        XtensaRsrWsr,
    }

    /// Probe which OpenOCD command form works for SR `name`.
    ///
    /// Tries `reg <name>` first, then `xtensa rsr <name>`.  Returns the first
    /// form whose response parses to a hex value; returns an error if none work.
    pub fn probe_sr_form(&mut self, name: &str) -> Result<SrAccessForm> {
        // Try `reg <name>` first.
        if let Ok(resp) = self.tcl(&format!("reg {}", name)) {
            if parse_hex_from_response(&resp).is_ok() {
                return Ok(SrAccessForm::Reg);
            }
        }
        // Try `xtensa rsr <name>`.
        if let Ok(resp) = self.tcl(&format!("xtensa rsr {}", name)) {
            if parse_hex_from_response(&resp).is_ok() {
                return Ok(SrAccessForm::XtensaRsrWsr);
            }
        }
        Err(anyhow!("probe_sr_form: no working command form for SR '{name}'"))
    }

    /// Read SR `name` using the cached or probed access form.
    ///
    /// On first call for a given SR, probes both forms via `probe_sr_form`.
    /// Subsequent calls reuse the cached form.
    pub fn read_sr(&mut self, name: &str) -> Result<u32> {
        let form = self.sr_form(name)?;
        match form {
            SrAccessForm::Reg => self.read_register(name),
            SrAccessForm::XtensaRsrWsr => {
                let resp = self.tcl(&format!("xtensa rsr {}", name))?;
                parse_hex_from_response(&resp)
                    .with_context(|| format!("read_sr({name}) xtensa rsr: {resp:?}"))
            }
        }
    }

    /// Write `val` to SR `name` using the cached or probed access form.
    pub fn write_sr(&mut self, name: &str, val: u32) -> Result<()> {
        let form = self.sr_form(name)?;
        match form {
            SrAccessForm::Reg => self.write_register(name, val),
            SrAccessForm::XtensaRsrWsr => {
                self.tcl(&format!("xtensa wsr {} 0x{:08x}", name, val))?;
                Ok(())
            }
        }
    }

    /// Internal: return the cached or freshly probed `SrAccessForm` for `name`.
    fn sr_form(&mut self, name: &str) -> Result<SrAccessForm> {
        if let Some(&form) = self.sr_forms.get(name) {
            return Ok(form);
        }
        let form = self.probe_sr_form(name)?;
        self.sr_forms.insert(name.to_string(), form);
        Ok(form)
    }
```

### Task 2: Windowed-register translation (HW root cause #2)

**Files:**
- Modify: `crates/hw-oracle/src/openocd.rs` (add `read_logical_ar` / `write_logical_ar`)
- Modify: `crates/hw-oracle/src/lib.rs:921-994` (route AR writes through the new layer + reorder so WB is set first)

**Background:** When a test calls `st.write_windowbase(4)`, the runner zeroes registers (lines 921-929) with WB=0 (still the boot default), then writes the test's setup registers (lines 964-967) — also still with WB=0 — and only then sets WB=4 (line 977-979). After that point, `read_register("a3")` reads physical AR15 (because WB=4, and a3 = (4*4 + 3) & 0x3F = 19). The setup writes to a3 went to physical AR3, which is now inaccessible without changing WB. Result: every windowed-test where `init_windowbase` is non-zero sees garbage in its setup registers.

The fix has two parts:
1. **Reorder operations** in `capture_hw_state` so WB is set BEFORE register writes that depend on it.
2. **Add explicit logical AR helpers** in `openocd.rs` that read current WB and route through it. This is for end-state capture, where reading `a0..a15` after execution should return the active frame's logical view (already correct via `oc.read_register("aN")` with the running WB), but doing it through an explicit helper documents intent and shields the call sites from future WB drift.

```rust
    /// Read logical AR `idx` (0..=15) through the current `WindowBase`.
    ///
    /// Computes `phys = ((wb << 2) + idx) & 0x3F` and reads physical AR.
    pub fn read_logical_ar(&mut self, idx: u8) -> Result<u32> {
```

```rust
    // ── Windowed-register translation (Plan 1.5 fix #2) ───────────────────────

    /// Read logical AR `idx` (0..=15) through the current `WindowBase`.
    ///
    /// Temporarily rotates WB so that the desired physical slot maps to a0..a3,
    /// reads, then restores the original WB.
    pub fn read_logical_ar(&mut self, idx: u8) -> Result<u32> {
        assert!(idx < 16, "read_logical_ar: idx {idx} out of range 0..=15");
        let wb_orig = self.read_register("windowbase")?;
        let phys = ((wb_orig << 2) + idx as u32) & 0x3F;
        let wb_temp = phys >> 2;
        let logical_in_temp = (phys & 0x3) as u8;
        self.write_register("windowbase", wb_temp)?;
        let val = self.read_register(&format!("a{logical_in_temp}"))?;
        self.write_register("windowbase", wb_orig)?;
        Ok(val)
    }

    /// Write `val` to logical AR `idx` (0..=15) through the current `WindowBase`.
    pub fn write_logical_ar(&mut self, idx: u8, val: u32) -> Result<()> {
        assert!(idx < 16, "write_logical_ar: idx {idx} out of range 0..=15");
        let wb_orig = self.read_register("windowbase")?;

        self.write_register("windowbase", wb_temp)?;
        self.write_register(&format!("a{logical_in_temp}"), val)?;
```

### Task 9: MOVSP register spill

**Files:**
- Modify: `crates/core/src/cpu/xtensa_lx7.rs:1038-1053` (`Movsp { at, as_ }` arm)
- Test: `crates/hw-oracle/tests/oracles.rs` (new oracle test)

**Background:** When MOVSP detects a live adjacent frame (`WindowStart[(WB+1) & 0xF] == 1`), it must spill the next frame's registers to memory at `[a[at] - 16, a[at])` per the windowed ABI before performing the move. Plan 1 raised AllocaCause instead. The fix implements the full spill, then completes the move.

Reference: ESP-IDF `xtensa-rt`'s `_WindowSpill` macro (MIT-licensed, design doc §7.6 reference rule allows reading).

- [ ] **Step 1: Read the reference spill semantics**

The canonical spill sequence per Xtensa ABI:
1. The "next" frame (WB+1) holds 4 logical AR slots (a0..a3 of that frame, which are physical AR (WB+1)*4 .. (WB+1)*4+3).
2. The new SP (in `a[at]`) is the destination's stack pointer; the old SP is in `a[as_]`.
3. The spill writes physical AR (WB+1)*4 .. (WB+1)*4+3 to memory at `[a[at] - 16, a[at] - 12, a[at] - 8, a[at] - 4]` respectively. This is per-Xtensa convention: the next frame's a0..a3 are saved into the destination stack frame's "spill slots" at negative offsets from the new SP.
4. After spill, the simple register move proceeds: `a[at] = a[as_]` (in current WB).
5. The `WindowStart[(WB+1) & 0xF]` bit is cleared (the frame is now spilled to memory and no longer live in the register file).

The plan implements exactly this.

- [ ] **Step 2: Write the failing test (MOVSP with live adjacent frame, requiring spill)**

```rust
///   a4 (current SP, will be moved into a3 = new SP) = SP_NEW = some safe scratch addr.

///     ar4 = 0xAAAAAAAA, ar5 = 0xBBBBBBBB, ar6 = 0xCCCCCCCC, ar7 = 0xDDDDDDDD.
///

///   - a3 (logical, current WB=0) = SP_NEW.
///   - WS bit 1 cleared.
///   - Memory at [SP_NEW - 16 .. SP_NEW - 4] = the four sentinels.

    // SP_NEW = IRAM_BASE + 0x1100 (DRAM scratch with at least 16 bytes preceding).
```

```rust
            // MOVSP at, as_ — Plan 1.5 full spill implementation.
            //
            // ISA RM §8 MOVSP semantics:
            //   if WindowStart[(WB+1) & 0xF] == 1 (adjacent frame live):
            //     spill physical AR ((WB+1)*4 .. (WB+1)*4+3) to memory at
            //     [a[at] - 16, a[at] - 4] respectively (Xtensa windowed ABI).
            //     Clear WindowStart[(WB+1) & 0xF].
            //   Then unconditionally: a[at] = a[as_]; PC += len.
            //
            // Reference: ESP-IDF xtensa-rt _WindowSpill macro (MIT-licensed,
            // design doc §7.6 reference rule).
            Movsp { at, as_ } => {
                let wb = self.regs.windowbase();
                let next_idx = wb.wrapping_add(1) & 0x0F;
                let new_sp = self.regs.read_logical(as_);

                if self.regs.windowstart_bit(next_idx) {
                    // Spill the next frame's physical AR slots to memory
                    // at [new_sp - 16, new_sp - 12, new_sp - 8, new_sp - 4].
                    let next_phys_base = (next_idx as usize) * 4;
                    for i in 0..4u32 {
                        let phys = (next_phys_base + i as usize) & 0x3F;
                        let val = self.regs.physical(phys);
                        let addr = new_sp.wrapping_sub(16).wrapping_add(i * 4) as u64;
                        bus.write_u32(addr, val)?;
                    }
                    // Clear the live bit for the now-spilled frame.
                    self.regs.set_windowstart_bit(next_idx, false);
                }

                self.regs.write_logical(at, new_sp);
                self.pc = self.pc.wrapping_add(len);
            }
```

```markdown

## Plan 1.5 runner additions

The HW runner gained three named capabilities in Plan 1.5:

1. **Per-SR access-form dispatch.** Some Xtensa SRs require `xtensa rsr <name>` /
   `xtensa wsr <name>` rather than `reg <name>`. `OpenOcd::read_sr` / `write_sr`
   probe both forms once per SR per session and cache the working one. Setup
   closures' SR writes (via `OracleState::write_sr` and helpers like
   `write_epc1`, `write_intenable`, etc.) all route through the dispatch.
2. **Windowed-register translation.** `OpenOcd::read_logical_ar(idx)` and
   `write_logical_ar(idx, val)` compute `phys = ((wb << 2) + idx) & 0x3F`
   from the current `WindowBase` and route to the corresponding physical AR.
   `capture_hw_state` calls these for end-state register capture.
3. **VECBASE write sequencer.** `OpenOcd::set_vecbase(addr)` wraps a halt →
   write → halt sequence so the next exception dispatch picks up the new
   value. `capture_hw_state`'s SR setup loop special-cases VECBASE through
   this method.

The runner also forces `WindowBase` / `WindowStart` BEFORE applying the
test's logical-AR setup writes (Plan 1.5 fix #2), so tests that pre-load
`WindowBase` via `OracleState::write_windowbase` see their setup registers
land in the correct physical slot.
```
