# Tier-1 Validation Matrix: repair + peripheral coverage expansion

Date: 2026-06-27
Branch: `feat/tier1-peripheral-coverage` (off `origin/main`)
Surface: https://app.labwired.com/validation → `docs/coverage/tier1-matrix.json`

## Problem

The public /validation matrix is generated by running per-chip Tier-1 fixture
firmware (`examples/tier1-fixture/<chip>/`) on the simulator and parsing
`TIER1 <class> PASS|FAIL` / `TIER1 done` lines over UART
(`crates/cli/src/tier1.rs`). The nightly job re-runs every fixture and commits
the refreshed JSON, gated by a ratchet that forbids `pass -> {blocked,partial}`
regressions.

Two things are wrong as of today:

1. **The matrix is stale.** Last nightly refresh was 2026-06-11 (16 days ago).
   The `stm32h563` fixture already proves all 12 universal classes + `can`, but
   the JSON still records adc/spi/pwm/rtc/irq as `na`. Conversely several chips
   the JSON records as `pass` no longer reproduce (see #2), so the ratchet has
   correctly *frozen* the page rather than laundering the regressions — leaving
   it stuck at stale-green.

2. **Silicon-fidelity features broke existing fixtures.** The real-RAM-size and
   per-peripheral RCC clock-gating features (landed after 2026-06-11) made the
   models stricter; the older fixtures predate them and now fail:
   - `stm32f103`: `memory.x` declared RAM = 128K but `stm32f103.yaml` models
     20KB → initial SP above modeled RAM → stack overflow on first PUSH, zero
     output. **Fixed:** RAM = 20K. (Verified: clock/gpio/timer/dma PASS.)
   - `stm32l476`: `stm32l476.yaml` clock-gates USART2 (`apb1enr1` bit 17) and the
     L4 RCC model gates the SYSCLK switch on the target source being ready. The
     fixture never enabled the USART2 clock (silent console) and selected HSI16
     as SYSCLK without turning HSI16 on (clock-sws FAIL). **Fixed:** enable
     USART2EN before first report; enable HSI16 + wait HSI16RDY before SW=01.
     (Verified: clock/gpio/timer/dma/irq PASS.)
   - `nrf52840`, `rp2040`: produce no output on current main; root cause TBD
     (Nordic/RP have no RCC gating — separate diagnosis).

## Goal

Maximize honest Tier-1 coverage (user picked Tier 1+2+3) and restore the matrix
to a truthful, reproducing state. Deliver as PRs to `labwired-core` main.

## Mechanism (unchanged)

A cell turns green only by the fixture proving it on the sim:
`check_<class>()` pokes the modeled registers, returns `Ok`/`Err`, and `report`
prints `TIER1 <class> PASS|FAIL`. Build via `scripts/tier1/build_*.sh`,
regenerate via `labwired tier1-matrix` + `generate_tier1_scoreboard.py`, refresh
`tests/fixtures/tier1/MANIFEST.json`.

Cell vocabulary: `pass` / `partial` / `blocked` / `na` (chip YAML declares no
such peripheral) / `unrecorded` (declared but no check, or no fixture).

## Plan (value-ordered, each tier independently reviewable)

### Tier 0 — repair + refresh (this PR)
Fix the broken fixtures (f103 ✓, l476 ✓, nrf52840, rp2040), rebuild all blobs,
regenerate the matrix + scoreboard, refresh MANIFEST. Net effect: the page
reproduces again and picks up h563's already-implemented checks.

### Tier 1 — fill the 28 modeled-but-unchecked cells
Add `check_*` for declared-but-`unrecorded` classes. Current inventory:
- stm32f103: i2c, spi, adc, wdt, rtc
- stm32f401: i2c
- stm32f407: timer, i2c, spi, adc
- stm32l073: i2c, spi, adc, wdt, rtc
- stm32l476: i2c, spi, adc, wdt, rtc
- nrf52840: clock, timer, i2c, spi, adc, pwm, wdt, rtc

### Tier 2 — YAML completeness (false `na` → checkable)
Several chips declare almost nothing despite near-identical silicon to passing
siblings (nrf52832: only uart; rp2040: only uart; esp32c3 partially). Audit each
chip YAML, declare peripherals the sim **already models**, then add checks.

### Tier 3 — new sim peripheral models
For genuinely-unmodeled `na` (e.g. esp32/esp32c3 i2c/spi/adc/pwm, rp2040
timer/pwm), write the peripheral model in `crates/core/src/peripherals/`, wire
into the SoC factory, declare in YAML, then check.

## Honesty guardrails (standing rules)

- A check earns PASS only if the model genuinely behaves under register pokes.
  If a poke reveals a stub, leave the cell `unrecorded`/`blocked` and record the
  gap — never fake a green.
- Every change is verified by *running* the fixture on the sim and confirming
  `TIER1 <class> PASS` + `done`, not by editing JSON.
- The ratchet test (`tier1_matrix_ratchet`) must stay green: no `pass ->`
  regressions.
- `memory.x` sizes must match the chip YAML (the f103 bug came from a comment
  that claimed they matched while they didn't).

## Out of scope

- Hardware-in-the-loop / silicon-anchored tier (separate workstream).
- The `feat/model-validation-report` branch (diverging CPU model; not a base).
